# Tier 2: 議事録 CMS・例規サイトのベンダーアダプター

> **執筆予定**

自治体ごとに異なる議事録検索 CMS と例規サイトを検出し、ベンダー別アダプターで取得・正規化するための設計文書です。全国共通の URL や DOM 構造を仮定せず、検出結果と適用したアダプターを provenance に残し、取得不能時は人による確認へ戻せる構成を示します。

## 章立て

- ベンダー検出のシグナルと誤検出防止
- アダプター共通インターフェース
- robots.txt、利用規約、取得間隔
- 会議、発言者、発言、ページ位置の正規化
- 例規の公布日、施行日、改廃履歴
- 原本キャッシュと差分更新
- SQLite/FTS5 とコンテキストパック
- 失敗時の隔離、再試行、人手確認
- テスト用フィクスチャとベンダー変更検知
