# Tier 1: 全国共通の統計・財政データ

Tier 1 は、Tier 0 で確定した自治体コードを使い、全国共通の公式ソースから基礎統計と財政指標を取得する。指標名だけで値を集めず、表の意味、対象時点、単位、原典を同時に確定する。

## 取得方針

すべての指標は、次の4点を一組として保存する。

| 項目 | 内容 |
|---|---|
| `value` | 数値または欠測。表示用の生文字列も別に保持する |
| `as_of` | 調査基準日、対象年度、または期間 |
| `definition` | 分母、会計範囲、対象人口など、値の意味 |
| `source` | 公式機関名、閲覧 URL、機械取得ロケータ |

これに `unit`、`raw_value`、`fetched_at`、原本ハッシュ、検証状態を加える。出典 URL だけでは、年度や定義の違う値を区別できない。

## 国勢調査: e-Stat API 3.0

機械取得には、e-Stat API 3.0 の三つの操作を使う。

```text
https://api.e-stat.go.jp/rest/3.0/app/json/getStatsList
https://api.e-stat.go.jp/rest/3.0/app/json/getMetaInfo
https://api.e-stat.go.jp/rest/3.0/app/json/getStatsData
```

公式仕様:

```text
https://www.e-stat.go.jp/api/api-info/e-stat-manual3-0
```

Tier 0 の地域メタ情報 API と異なり、e-Stat API 3.0 の利用には AppId が必要である。e-Statへユーザー登録した後、マイページの「ユーザ情報変更」→「登録内容変更」→「利用する機能」で「API機能」にチェックを入れ、画面下部の「変更」で確定する。その後、「API機能（アプリケーションID発行）」からAppIdを発行する。ユーザー登録だけではAPI機能が有効になっていない場合がある。

AppId は環境変数などの秘密情報として扱い、ログ、保存 URL、`authority_map.yaml` に書かない。`STATUS 100`またはHTTP 403の認証エラーでは、AppIdの誤入力だけでなく、「API機能」の有効化と変更確定も確認する。

### 統計表を意味から選ぶ

国勢調査の政府統計コード `00200521` を条件に `getStatsList` を検索し、次の順に表を選ぶ。

1. 応答内で重複する統計表を統計表 ID で除去する。
2. 集計区分と表題を照合し、必要な指標の意味に一致する候補だけを残す。
3. 類似語を含む別集計を除外する。地域区分、補完値、都市計画区分などが目的と異なる表は採用しない。
4. 必要な全指標が同じ `SURVEY_DATE` で揃う年月を求め、その中の最新を選ぶ。
5. `getMetaInfo` の分類名と分類項目から、指標、地域、単位に対応するコードを動的に選ぶ。
6. `getStatsData` の `cdArea` には5桁標準地域コードを渡す。
7. 意味条件ごとに値が一件になることを確認する。

「最も新しい人口」と「最も新しい年齢構成」を別々の年から混ぜない。新しい速報表が一部指標しか含まない場合、「要求した指標が同一基準日で揃う最新」を採用条件とする。

### 表 ID と文字列条件

API の表題や分類名は、永続的なプログラム契約ではない。通常はメタ情報から動的に探索し、既知の全国表 ID は探索規則が壊れた場合の限定的なフォールバックにとどめる。

検証済みスパイクで確認した全国表 ID は次のとおりである。これらは自治体別の値ではなく、全国共通の統計表を指す。ただし、特定の調査回に固定されたIDであるため、通常経路の「最新表」としては使わない。

| 指標 | 全国表 ID |
|---|---|
| 総人口 | `0003445078` |
| 総世帯数 | `0003445098` |
| 65歳以上人口構成比 | `0003445163` |

フォールバックを使ったときは、次を守る。

- 「最新」という表示を外す
- 採用した基準日を明示する
- 表題、分類、単位、対象地域コードをメタ情報で再検証する
- フォールバック理由を検証記録に残す
- 個別自治体の値をコードや設定へ埋め込まない

公開出典には `https://www.e-stat.go.jp/dbview?sid=<statsDataId>` のような閲覧用 URL を保存し、AppId を含む API リクエスト URL は保存しない。

### 定義を推測しない

公表済みの比率を、別途取得した人口から安易に再計算しない。年齢不詳の扱いなど、分母の定義がメタ情報だけで確定できないことがあるためである。定義が十分でなければ、公式公表値を保持しつつ `definition_status: needs_review` とする。

`SURVEY_DATE` が年月だけを持つ場合、日付を補完するには調査固有の基準日規則が必要になる。別統計へ一般化せず、補完根拠を `definition` または provenance に記録する。

## 財政: 総務省「市町村別決算状況調」

財政指標の第一候補は、総務省「市町村別決算状況調」の概況 XLSX である。

一覧ページ:

```text
https://www.soumu.go.jp/iken/kessan_jokyo_2.html
```

概況 XLSX は自治体ごとに一行で整理され、6桁地方公共団体コードで直接照合できる。財政力指数、経常収支比率、実質公債費比率、将来負担比率、歳入総額、歳出総額など、基礎的な比較に必要な項目を同じ行から取得できる。

列記号や列番号を固定しない。年度ごとに見出し行を読み、正規化した見出し名から列を解決する。自治体名だけではなく6桁コードで結合し、名称は照合用に使う。

### 発見の手順

1. 一覧ページを取得する。
2. 掲載リンクから対象年度の市町村ページを発見する。
3. 年度ページ内で、都市・町村など対象区分に合う「概況」ファイルを発見する。
4. リンク先の拡張子、応答の Content-Type、ファイルシグネチャを検証する。
5. XLSX の見出しと6桁コード列を特定する。
6. 対象コードの行が一件であることを確認する。
7. 必要列を抽出し、単位と注記を記録する。

> [!CAUTION]
> `https://www.soumu.go.jp/main_content/<opaque-id>.xlsx` の `<opaque-id>` は、年度、自治体コード、資料種別から導出できない。検証済み URL であっても固定値として実装せず、必ず一覧ページと年度ページから現行リンクを発見する。

HTML の URL に年度らしい規則があっても、番号を組み立てて直行しない。索引のリンクを公式な探索起点にする。年度切替直後には、ページと XLSX の公開時刻がずれる可能性があるため、対象年度のファイルが揃わない場合は「未公開」として停止する。前年度へ戻すなら、利用者へ明示し、`as_of` を書き換える。

## 検算経路: 決算カード

総務省「決算カード」は、概況 XLSX の値を人が確認し、抽出結果を照合する第二候補とする。

索引ページ:

```text
https://www.soumu.go.jp/iken/zaisei/card.html
```

索引から最新の市町村決算カード年度ページを発見し、対象都道府県の XLSX と PDF へのリンクをたどる。現行資料は、都道府県単位のファイルに複数自治体が含まれる形がある。5桁コードの先頭2桁で都道府県を選び、XLSX 内では自治体シートまたは自治体見出しを一意に照合する。

決算カードも `main_content` ID を使うため、ファイル URL を推測できない。過去資料に見られるファイル名規則を現行資料へ流用しない。

ラベル近傍から値を抽出する方式は、シート配置の変更に弱い。機械抽出した値は PDF 表示と照合可能にし、一意なラベルと値セルを選べない場合は検証待ちにする。

## 欠測、会計範囲、単位

財政資料の `-` などの記号は、ゼロへ変換しない。

```yaml
value: null
raw_value: "-"
verification_state: "needs_definition_review"
```

欠測記号が「算定なし」「該当なし」「閾値未満」のどれを意味するかは、当該年度の注記で確認する。意味が確定するまで、公開文章で理由を断定しない。

また、歳入総額・歳出総額が普通会計など特定の会計範囲を示す場合、自治体の全会計を単純合算した額として説明してはならない。金額の単位は、表見出し、注記、セル書式の順に確認し、表示時の換算処理も記録する。

## 補助経路: e-Stat の地方財政ファイル

e-Stat の統計ファイル一覧も、地方財政データの補助的な探索経路になる。

```text
https://www.e-stat.go.jp/stat-search/files?kikan=00200&layout=dataset&toukei=00200251
```

最新版が REST API の表として提供されず、ファイル一覧に CSV/XLSX として掲載される場合がある。ダウンロード URL を固定せず、一覧から統計表、対象年度、ファイル種別を発見する。

CSV の実体が CP932 である場合があるため、HTTP ヘッダーだけで文字コードを決めない。UTF-8 として黙って置換文字を入れるのではなく、デコード候補を検証し、見出しと団体コードの形式が正常に読めることを確認する。

## 保存モデル

指標行は、少なくとも次を保持する。

```yaml
indicator:
  key: "<stable-indicator-key>"
  value: null
  raw_value: "<source-cell-text>"
  unit: "<unit>"
  as_of: "<date-or-fiscal-year>"
  definition: "<scope-and-denominator>"
  source_name: "<official-source-name>"
  source_url: "<human-readable-official-url>"
  source_locator:
    kind: "<api-or-xlsx>"
    table_id: "<optional-table-id>"
    sheet: "<optional-sheet-name>"
    row_key: "<municipality-code>"
    column: "<normalized-header>"
  fetched_at: "<ISO-8601 timestamp>"
  verification_state: "<state>"
```

`authority_map.yaml` は、この行を用途別に指す。議会質問、住民向け説明、内部分析では必要な厳密さや推奨原典が異なり得るため、`indicator × use_case` で裁定する。

## 最低限の再検証

別環境、別年度、公開元の更新後には、次を再確認する。

- e-Stat API 3.0 の三操作が成功する
- 必要指標の最大共通 `SURVEY_DATE` が意図した調査回になる
- 対象5桁コードが各表の地域分類に存在する
- 意味条件ごとに値が一件になる
- 総務省の一覧から年度ページと概況 XLSX を発見できる
- 6桁コードの対象行が一件になる
- 決算カードまたは注記で定義と単位を確認できる
- `PRAGMA integrity_check` が `ok` になる
- オフライン再構築で同じ原本から同じ行を生成できる

## 実装

Tier 1a は `bootstrap/cli/census.py`、Tier 1b は `bootstrap/cli/fiscal.py`、Open XMLの最小読取器は `bootstrap/cli/xlsx.py` に実装した。

```bash
export ESTAT_APPID='e-Statから発行されたAppId'
python3 -m bootstrap.cli '自治体名'
python3 -m bootstrap.cli '自治体名' --cross-check
python3 -m bootstrap.cli '自治体名' --offline --cross-check
```

国勢調査は、3種類の意味検索を行い、表IDの重複、集計区分、表題、除外語を検査する。必要な3指標が共通して持つ `SURVEY_DATE` の最大値を選び、各表の `getMetaInfo` から分類コードを解決して、対象5桁コードの値が意味条件ごとに一件になることを確認する。

動的探索または選択表のメタデータ照合に失敗した場合だけ、検証済み2020年表 `0003445078`、`0003445098`、`0003445163` を一組で使う。この場合、実行レポートと `authority_map.yaml` に `latest-ness lost` の警告と理由を残す。公開URLは `https://www.e-stat.go.jp/dbview?sid=<statsDataId>` とし、AppId入りAPI URLを保存しない。

財政の主経路は、一覧ページから最新年度ページ、年度ページから自治体種別に合う「概況」XLSXを発見する。`main_content` IDはコードに固定しない。XLSXはZIP/XMLとして読み、共有文字列・インライン文字列・ワークブック関係を解決する。列番号は固定せず、団体コード、団体名、6指標の見出しラベルから列を決める。既知の別名へフォールバックした場合は、採用ラベル、ヘッダーセル、`header_fallback_used` を来歴に残す。

対象行は6桁団体コードで一意に選び、団体名も照合する。欠測記号は `value = NULL`、`raw_value = <原記号>` として保持する。`--cross-check` を指定した場合だけ、決算カード索引から同年度ページと対象都道府県XLSXを発見し、自治体シートの6指標を比較する。決算カード取得・抽出の失敗は主経路の値へ置き換えず、任意検算の失敗として警告する。

### 検証状況

2026-07-23、太良町で次をライブ確認した。

- `api.e-stat.go.jp/robots.txt` と `www.soumu.go.jp/robots.txt` を各取得前に確認し、必要経路が許可されていた
- e-Stat API 3.0 の `getStatsList`、`getMetaInfo`、`getStatsData` が成功
- 動的探索が、3指標の揃う最新共通 `SURVEY_DATE` として2020年10月を選択。2020年固定表へのフォールバックは不使用
- 総務省一覧から令和6年度ページを発見し、町村概況XLSXを発見
- 6桁団体コードと団体名で対象行を一意に照合し、6指標を見出しラベルから抽出
- 同年度の決算カードXLSXを索引から発見し、6指標すべてが主経路と一致
- SQLite 9指標行と `PRAGMA integrity_check = ok`
- オフライン再実行はHTTPリクエスト0件で、オンライン実行と同じ9指標行を再構築
- AppId実値の保存がないことをキャッシュと出力全体で検査

ライブ取得は実装・修正中を含めて累計21リクエストだった。全自治体種別、他都道府県、過年度、次年度のレイアウト、動的探索失敗時の実APIフォールバック、新設・再編自治体、決算カードの全欠測注記は未検証である。
