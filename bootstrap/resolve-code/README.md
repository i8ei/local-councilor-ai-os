# Tier 0: 自治体名から標準地域コードを解決する

Tier 0 は、利用者が入力した自治体名を、現行自治体の5桁標準地域コードへ変換する。主経路は、AppId を必要としない公式の「統計ダッシュボード WebAPI・地域メタ情報」である。

## 主経路

地域メタ情報 API:

```text
https://dashboard.e-stat.go.jp/api/1.0/Json/getRegionInfo
```

自治体名による候補検索:

```text
?Lang=JP&RegionLevel=8,9,10,12,13&SearchRegionWord=<URLエンコードした自治体名>
```

都道府県表:

```text
?Lang=JP&RegionLevel=3
```

公式仕様:

```text
https://dashboard.e-stat.go.jp/static/api
```

対象の地域レベルは次のとおりである。

| 地域レベル | 対象 |
|---:|---|
| 8 | 政令指定都市 |
| 9 | 市 |
| 10 | 東京都特別区 |
| 12 | 町 |
| 13 | 村 |

地域レベル11の政令指定都市の行政区は、独立自治体ではなく市町村財政データの照合単位にもならないため、既定では除外する。

## 解決手順

1. 入力名と API 応答の地域名を Unicode NFC に正規化する。
2. API の部分一致結果から、正規化後の名称が完全一致する候補だけを残す。
3. `@toDate == "999912"` の候補を現行団体として扱う。
4. 候補が一件なら、その5桁コードを採用する。
5. 候補が複数なら、都道府県ヒントで絞る。ヒントがなければ候補一覧を示して停止する。
6. 候補がなければ、表記揺れの訂正を求めるか、代替経路へ進む。

部分一致の先頭候補を自動採用してはならない。同名自治体や、名称の一部が一致する別団体を誤採用する危険がある。

## 都道府県名の解決

町村の地域階層では、直接の親要素が郡になることがある。このため、親要素をそのまま都道府県名として使わない。

5桁標準地域コードの先頭2桁を都道府県コードとして取り出し、地域レベル3の都道府県表と照合する。入力された都道府県ヒントも、同じ表にある正式名称へ正規化してから比較する。

## 5桁と6桁を混同しない

- e-Stat の標準地域コード: 5桁
- 総務省の地方公共団体コード: 検査数字を含む6桁

財政ファイルとの結合では6桁コードが必要になる。5桁コードから検査数字を計算する場合も、配布元の6桁コードとの一致を検証し、導出値だけを無条件に信用しない。

データベースでは、次のように意味を分けて保存する。

```yaml
municipality:
  area_code_5: "<5-digit-code>"
  local_government_code_6: "<6-digit-code>"
  name: "<municipality-name>"
  prefecture_code_2: "<2-digit-code>"
  resolved_from: "e-Stat dashboard getRegionInfo"
  resolved_at: "<ISO-8601 timestamp>"
```

## 代替経路

主経路が利用できない場合は、総務省「全国地方公共団体コード」の配布ページを入口にする。

```text
https://www.soumu.go.jp/denshijiti/code.html
```

このページから、現在掲載されている XLSX へのリンクを発見して取得する。XLSX には検査数字を含む6桁コードが掲載されるため、名称と都道府県を照合したうえで、先頭5桁を e-Stat 用コードとして利用できる。

> [!WARNING]
> XLSX の実体 URL に含まれる `main_content` ID は不透明であり、更新時に変わる。検証時のファイル URL を固定せず、必ず配布ページからリンクを発見する。

XLSX を読むために大規模な表計算ライブラリを必須にする必要はない。実行環境を小さく保つ場合は、Open XML を ZIP と XML として読み、共有文字列、セル値、シート関係を解決できる。ただし、書式だけで意味を推測せず、見出し名称とコード形式を検証する。

## 失敗として停止すべき条件

- 完全一致候補が複数あり、都道府県で一意に絞れない
- 現行団体の判定情報が欠けている
- 5桁コードと都道府県表が矛盾する
- 主経路と代替経路のコードが一致しない
- 配布ページから現行 XLSX を発見できない
- XLSX 内で名称と都道府県の組み合わせが一意でない

これらは推測で埋めず、人へ確認を返す。合併、編入、新設直後の団体では、過年度統計に現行コードが存在しないことがあるため、コード解決の成功と過年度データの取得成功は別々に記録する。

## 検証記録

解決結果には、少なくとも次を残す。

- 入力された名称と都道府県ヒント
- 正規化後の名称
- 候補件数と採用理由
- 5桁標準地域コードと6桁地方公共団体コード
- API または配布ページの URL
- 取得時刻
- 代替経路との照合状態
