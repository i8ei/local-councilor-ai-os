# 全国基礎自治体registry

Tier 0がネットワークなしでも自治体名、都道府県、5桁標準地域コード、6桁全国地方公共団体コード、公式ホームページ入口を確定できるよう、現在の基礎自治体だけを小さなCSVとして同梱する。

## 収録条件

一つの公開元だけを無条件に採用せず、次の三経路すべてで一致した団体を収録する。

- e-Stat統計ダッシュボード地域メタ情報: 現行期間、5桁コード、地域レベル
- J-LIS「地方公共団体コード住所」: 6桁全国地方公共団体コードと団体名
- J-LIS「全国自治体マップ検索」: 地方公共団体公式ホームページへの掲載リンク

この共通部分を取ることで、政令指定都市の行政区、境界未定地域、公式ホームページを持たない地域区分を基礎自治体として混入させない。e-Statの現行地域表にあるが収録条件を満たさない地域は、理由とともに`metadata.json`へ列挙する。e-StatとJ-LISで字体が異なる名称は、5桁コードとJ-LIS内のコード・ホームページ表記が一致することを条件に、e-Stat名を正規名、J-LIS名を検索aliasとして両方残す。J-LISの一括ダウンロード専用サービスにはアクセスせず、一般公開されている都道府県別索引だけを低頻度で読む。

`municipalities.csv`は実行時の解決用スナップショット、`metadata.json`は各取得ページのURL、取得時刻、SHA-256、cache利用状態を記録する来歴である。CSVのURLはJ-LIS掲載時点の値であり、接続先の現在の安全性や最新性を無条件に保証するものではない。

## 更新

通常はcacheを使って全体整合性と同梱CSVとの差分有無を確認する。`registry_changed: false`なら収録内容は同じである。

```bash
python3 -m bootstrap.municipalities.update
```

公式索引を再取得し、自治体数、コード検査数字、三経路の一致を確認した後にスナップショットを更新する。

```bash
python3 -m bootstrap.municipalities.update --refresh --write
```

更新は約100ページをホスト単位の低頻度で取得する。robots.txtを先に確認し、明示的な拒否やサーバー障害時は停止する。自治体再編等で件数が変わった場合は自動で受け入れず、差分と公式根拠を人が確認してから`--expected-count`を変更する。

## 実行時の扱い

`bootstrap.cli.resolve`は同梱registryを先に完全一致検索する。一意ならネットワークを使わず返し、同名自治体なら都道府県ヒントがない限り停止する。registryに存在しない名称だけ、従来のe-Stat APIへフォールバックする。

スナップショットの生成日時は「その時点で三経路を照合した日」であり、毎回のbootstrap実行時に最新性を再確認したことを意味しない。Tier 1の統計・財政資料の鮮度判定とも別である。
