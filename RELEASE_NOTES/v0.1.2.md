# v0.1.2

## Added

- Settlement review guidance that deliberately avoids pretending to solve arbitrary PDF extraction.
  - `csv_contract.md`
  - `extraction_guidance.md`
  - `failure_patterns.md`
  - `public_private_boundary.md`
- `insights.py` to generate human-review candidates only after `verify_totals.py` passes.
- `csv_templates.py` to print required CSV headers for `summary`, `revenue`, and `expenditure`.
- GitHub Actions workflow that runs `./run_tests.sh` on push and pull requests.

## Clarified

- PDF extraction is local, case-specific work by humans, AI tools, OCR, or text extraction.
- The repository owns the receiving contract, provenance, SQLite schema, reconciliation gate, common failure patterns, and public/private boundary.
- A passing reconciliation gate does not prove causes, policy evaluation, or public suitability.
