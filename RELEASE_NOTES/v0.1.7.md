# v0.1.7

## Added

- Added `modules/budget-review/` for budget review workflows.
- Added a budget SQLite schema and input contract for initial, supplemental, and current budgets.
- Added CSV-to-SQLite ingestion as an intermediate input path, with SQLite as the working database.
- Added budget verification gates:
  - revenue/expenditure balance
  - hierarchy totals
  - current-year vs previous-year comparison
  - pre-supplement + supplement = post-supplement
- Added budget review candidate generation after verification passes.
- Added budget failure patterns and a budget-to-settlement bridge guide.
- Added budget-review tests and included the module in `run_tests.sh`.

## Clarified

- The repository now covers both budget review and settlement review as separate modules.
- Budget PDF extraction remains local, case-specific work by AI, OCR, tools, or human review; the shared core is the SQLite input contract and verification gate.
