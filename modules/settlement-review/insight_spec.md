# 決算分析出力仕様

## 目的

v0.2 では、差額ゼロの検算を通った決算 DB から、9月決算審査で人が確認する分析候補を生成する。本書は出力契約だけを定め、分析処理はまだ実装しない。候補は問いの入口であり、重要性や原因を DB 内で断定しない。

## 入力条件

入力行の `verification_state` は `reconciled` とし、比較対象の会計範囲、階層、定義が一致していることを確認する。年度間で項目コードが変わる場合は、名称だけで自動結合せず、対応表と判断理由を別に残す。

予算書と決算書を結ぶ場合は、各行の `unit` を読む。原値を暗黙に変換せず、比較時に換算元、換算先、倍率を `transformations` へ記録する。

## 生成する候補

### 複数年推移

款、項、目、節の同一粒度について、予算現額、収入済額、支出済額、不納欠損額、収入未済額、翌年度繰越額、不用額の推移を出す。対象年度、欠落年度、コード変更、定義変更を併記する。増減率は分母がゼロまたは欠損なら計算しない。

### 新設項目と消滅項目

前年度に存在せず当年度に現れた行と、前年度に存在して当年度にない行を分ける。名称変更、階層移動、統合、分割の可能性を未解決事項として残し、直ちに新規事業や廃止事業と呼ばない。

### 大きな不用額

不用額の絶対額と予算現額に対する割合を候補指標にする。固定しきい値は設定ファイルへ置き、使用した値を出力へ残す。翌年度繰越額と混同せず、原因は決算書の数値だけから推定しない。

### 三段階比較

予算 DB の当初予算、決算 DB の予算現額、決算の収入済額または支出済額を並べる。補正後の予算現額との差と、最終的な執行差を分離する。単位換算、会計範囲、比較粒度が一致しない行は候補から隔離する。

## 出力契約

各分析候補は、少なくとも次を持つ。

| フィールド | 内容 |
|---|---|
| `candidate_id` | 再生成しても追跡できる識別子 |
| `analysis_kind` | `multi_year_trend`、`appeared`、`vanished`、`large_unused`、`three_stage` のいずれか |
| `statement` | 数値から直接言える最小の事実 |
| `value` | 原値または決定的な計算結果 |
| `as_of` | 対象年度または比較期間 |
| `definition` | 会計範囲、階層、式、除外条件 |
| `source` | 原典名、URL、印刷ページ、PDF ページ、元行ID |
| `transformations` | 単位換算と計算の順序 |
| `verification_state` | 入力と出力の検証状態 |
| `unresolved` | 定義差、名称変更、原因などの未解決事項 |

`value`、`as_of`、`definition`、`source` の4点セットは省略しない。複数値を比較する場合は、各値に4点セットを持たせる。出力に「問題」「改善」「浪費」などの評価語を自動付与しない。

## 証拠台帳と質問候補への受け渡し

各候補から、使用した元行ごとに根拠レコードを作り、`candidate_id` を `used_by` へ記録する。計算値は元行ID、式、単位換算から再構築できるようにする。証拠台帳で原典、時点、定義、単位、合計突合を確認した後、人が質問候補へ採用する。

質問候補には、確認済みの現状、行政へ説明を求める事項、未解決の仮説を分けて渡す。原因や妥当性は分析候補から自動確定しない。

## アンチパターン

- **前年差だけで質問を確定する。** 補正、繰越、定義変更、項目移動を確認できない。
- **名称一致だけで年度間結合する。** 同名で範囲が異なる行や、改称された同一項目を誤る。
- **単位を入力時に上書きする。** 原値との対応と換算過程が失われる。
- **不用額へ評価を付ける。** 数値から原因や政策的妥当性までは決まらない。
