# 02 現場受付

## 要旨

現場受付では、住民の声と現場観察を改変しない内部原典として受け取り、個人固有の事情を制度上の論点から切り離し、公開原典だけで支持も反証もできる問いへ変換する。匿名化した原文を公開側へ移す手順ではない。内部情報は課題発見のシグナルとして保持し、公開上の事実は別途取得した公式情報で組み立てる。

## 発動条件

住民から相談、要望、体験談を受け取ったとき、議員本人が窓口、施設、地域で運用上の違和感を観察したとき、提供文書や私的メモから制度上の課題候補が生じたときに発動する。個人を特定し得る内容がなくても、公開情報として扱えると決めつけず、この手順を通す。

## 入力と前提

- 受け取った言葉または観察の原記録
- 取得日、取得経路、本人との取扱上の約束
- 内部原典を保存できる管理領域
- [内部情報から公開可能な問いへ](../way-of-working/05-internal-to-public.md)と[安全境界](../principles/safety-boundary.md)を適用する前提
- 問いの登録先となる[論点ノート](../templates/issue-note.md)と接続先の案件

## 手順

1. **【人】原文を整えず内部原典として保存し、`visibility`、`contains_pii`、`source_type`、`verification`、`public_use`、`source_date`の六属性を付ける。**発言と議員本人の観察や評価を同じ原文に混ぜない。欠落する属性が一つでもあれば、`internal`、`unverified`、外部利用不可として止める。
2. **【人】この原典をクラウドAIへ入力するかを明示的に決める。**`contains_pii: true`の内容は、利用環境、目的、送信範囲を人が確認して承認しない限り入力しない。氏名を除いたこと、ローカルに保存したことは送信許可を意味しない。
3. **【併走】個人に固有の事情と制度上確認できる論点を分ける。**AIを使える入力だけなら分離案を作らせてよいが、人が原文と照合して確定する。日時、場所、家族構成、健康状態、職業、希少な出来事は、組合せによる再識別も考慮して公開側へ移さない。
4. **【併走】制度上の論点を、統計、法令や例規、予算決算、議事録、公開行政資料で答えられる問いへ変える。**主語を個人から制度、事業、窓口、地域全体へ移し、対象期間、対象範囲、判断基準、実績のどれを確かめるかを明示する。AIが候補と原典種別を出し、人が相談内容を過度に一般化していないか確認する。
5. **【人】接続を一方向に固定する。**内部原典には派生先を追跡する`source_id`を保持してよい。公開側の論点ノートには内部ID、内部ファイル名、原文の断片、内部ノートへのリンクを置かない。将来の訂正に必要な対応関係は内部側だけで管理する。
6. **【併走】確定した問いを[論点ノート](../templates/issue-note.md)へ登録する。**各問いに確認対象、候補となる公開原典種別、未確認事項を付け、既存案件へ接続する。案件がなければ[01 案件開始](01-start-case.md)を実行する。AIが形式と逆リンクの不在を点検し、人が登録内容を承認する。

## プロンプト断片

```text
以下は個人を特定できる情報を除いた制度上の論点です。公式公開情報で検証できる問いの候補を作り、各問いに対応する公開原典の種別を示してください。入力から除去し切れていない可能性がある属性や文脈は「非識別化できなかった要素」として別欄に列挙してください。個人の体験を地域全体の事実として一般化しないでください。

論点:
```

```text
次の問いを点検し、主語が個人ではなく制度または運用になっているか、公開原典だけで支持または反証できるか、対象期間と確認対象が明確かを判定してください。修正案には内部情報、識別子、原文の引用を含めないでください。
```

## 成果物と完了条件

六属性を備えた改変されていない内部原典が保存され、少なくとも一つの検証可能な問いが[論点ノート](../templates/issue-note.md)へ登録されていること。問いが案件へ接続され、公開側から内部原典への逆リンク、内部ID、再識別可能な断片が存在しなければ完了とする。

## 次のワークフローへの接続

案件がなければ[01 案件開始](01-start-case.md)で目的、権限、成果物を定める。案件と問いがそろったら[03 公開調査](03-public-research.md)へ進み、内部原典を根拠に使わず公開情報だけで検証する。

## アンチパターン

- **氏名を消せば公開可能と判断する。** 時期、場所、属性、希少な出来事の組合せで本人を推測できる。
- **原文を読みやすく整えて保存する。** 発言、観察、評価の境界が消え、後から事実関係を確かめられない。
- **相談件数を発生率として扱う。** 相談経路の偏りと未相談者を測れず、地域全体を代表しない。
- **内部原典を公開主張の出典にする。** 第三者が検証できず、本人の意向と安全境界を損なう。
- **公開側から内部側へリンクする。** 識別子を伏せても、リンク経路そのものが逆引きの手掛かりになる。
