# 05 内部情報から公開可能な問いへ（問い化）

## 要旨

問い化は、内部情報を匿名化して公開文へ移す処理ではない。住民の声や現場観察を課題発見のシグナルとして受け止め、個人の事情を切り離し、統計、制度、予算、議事録などの公開原典だけで検証できる問いへ組み替える処理である。公開成果物の根拠は最後まで公開情報に置き、内部原典との接続は内部側だけに残す。

## 内部原典は入口で属性を固定する

内部情報は、受け取った時点で原文と属性を一体にして保存する。後から公開可否を推測すると、相談時の約束も、未確認情報だった事実も失われる。

最低限、`visibility`（`internal` / `public`）、`contains_pii`、`source_type`（`official` / `field_observation` / `resident_input` / `provided_document` / `personal_note`）、`verification`（`unverified` / `corroborated` / `confirmed`）、`public_use`（`prohibited` / `anonymize_and_review` / `allowed`）、`source_date`を持たせる。属性が欠けた記録は「扱い不明」ではなく内部・未確認・外部利用不可として止める。

原文は都合よく整形しない。要約や論点メモを作る場合も、原文とは別の派生物として保存し、誰の発言か、事実確認済みか、議員自身の観察かを混ぜない。

## 私的に得た情報はシグナルであって権威ではない

住民の声と現場観察は、行政資料が捉えにくい摩擦を見つけるために使う。切実さや具体性は調査を始める理由になるが、公開された統計や制度上の事実を上書きする権威にはならない。

複数の相談が一致しても、それだけで発生率や制度全体の欠陥を断定しない。相談の偏り、未相談者の存在、時点差を測れないためである。そこで立てるのは結論ではなく、公開資料で支持も反証もできる仮説である。

## 人の話を制度の問いへ組み替える

最初に、内部原典から「個人に固有の事情」と「制度・運用として確認できる論点」を分ける。日時、場所、家族構成、疾病、職業、希少な出来事は、氏名がなくても再識別につながるため、論点側へ持ち込まない。

次に、論点を公開原典が答えられる形へ書き換える。主語を「ある住民」から制度、事業、窓口、地域全体へ移し、対象期間、対象者、判断基準、予算措置、実績、過去答弁のどれを確かめるかを明示する。

例：内部原典の「申請を何度もやり直した」という訴えを、その人の経緯として紹介しない。「当該制度では、申請不備の件数と主な理由を把握しているか。案内文、審査基準、補正手続は公開され、年度ごとの改善を検証できるか」と問い直す。

問いを立てた後は、法令・例規、予算決算、統計、行政資料、議事録から公開根拠を集める。この段階の主張管理は[06 証拠台帳](06-evidence-ledger.md)へ渡し、内部原典がなくても事実関係を再構成できる状態にする。

## 接続は内部から公開への一方向にする

公開ノート、質問原稿、説明資料から内部原典へはリンクしない。内部ID、ファイル名、個人用タグ、引用断片も、逆引きの手掛かりになるため外へ出さない。

内部側は、派生した公開上の問いや調査案件を追跡するために`source_id`を保持してよい。この接続は内部での説明責任と訂正のために使い、公開側へ転記しない。一方向性の詳しい安全境界は[安全境界](../principles/safety-boundary.md)に従う。

## 公開利用は人が文脈まで確認する

内部情報に由来する内容を少しでも公開利用する前に、人が再識別リスク、本人の意向、事実の正確性を確認する。再識別は直接識別子の有無だけでなく、状況、時期、地域、出来事の希少性、それらの組合せで判定する。

`contains_pii: true`の内容は、明示的な指示と利用環境の確認なしにクラウドAIへ読み込ませない。ローカルの保管庫へ保存した事実は、外部送信の安全性を一切保証しない。同期、貼り付け、検索連携、要約依頼も外部送信になり得る。

## 最小の始め方

1. 内部原典の入口に6属性を必須化し、欠落時は外部利用不可にする。
2. 一件の原典から個人事情を除き、公開資料で反証可能な問いを一つ作る。
3. 公開原典だけで問いの事実部分を組み立て、証拠台帳へ登録する。
4. 公開前に再識別、本人の意向、正確性、一方向リンクを人が確認する。

## アンチパターン

- **氏名を消せば公開できると考える。** 時期、場所、属性、希少な出来事の組合せで本人は推測される。
- **相談件数を地域全体の実態とみなす。** 相談経路の偏りがあるため、発生率や代表性の根拠にはならない。
- **内部原典を公開主張の出典にする。** 第三者が検証できず、本人の意向と事実確認も置き去りになる。
- **ローカル保存済みだからAI入力も安全とみなす。** 保存場所と送信先の安全条件は別であり、クラウドへの入力時に境界を越える。
