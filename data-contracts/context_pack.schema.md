# コンテキストパック契約

> **執筆予定**

SQLite/FTS5の検索層から、特定の問いに必要な根拠だけを選び、AIへ渡す小さなコンテキストパックの構造を定義する。原典やデータベースの全量投入を避け、公開・内部の区分、根拠ID、引用範囲、トークン予算、利用目的を機械的に検査できる契約を目指す。

## 構成予定

- パックID、目的、問い、作成日時
- 公開・内部の情報区分
- 根拠IDと原典位置
- 抜粋本文と前後文脈
- 検索条件と選択理由
- トークン・件数の上限
- 欠落情報と未解決事項
- AIへの許可事項と禁止事項
- 有効期限、破棄、再生成
- JSON Schemaと検証例
